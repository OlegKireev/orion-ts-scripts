# ⚔️ Скрипты для Orion Launcher (TypeScript)

Этот проект — удобная среда для написания сложных и надежных макросов для Ultima Online под Orion Launcher.

Вместо того чтобы писать огромные полотна кода в одном файле без подсказок, мы пишем скрипты на **TypeScript**. Это дает автодополнение в редакторе (VS Code), проверку ошибок до запуска и возможность разбивать код на модули. Затем специальная программа (сборщик) сама склеивает это в готовые `.oajs` файлы, понятные Ориону.

## 🚀 Быстрый старт (Установка)

Чтобы всё заработало, на **твоем** компьютере должны быть установлены [Node.js](https://nodejs.org/) (для сборки) и [Git](https://git-scm.com/) (для работы с кодом).

1. **Скачай** этот проект к себе на компьютер (через Git).
2. **Открой** папку с проектом в консоли (или прямо в терминале VS Code).
3. **Введи** команду для установки всех нужных инструментов:

```bash
npm install
```

## 📁 Как устроены папки

- 📂 **`src/scripts/`** — **Твои макросы лежат здесь.** Каждый файл в этой папке превратится в отдельный готовый макрос для Orion.
- 📂 **`src/lib/`** — **Общие библиотеки.** Здесь лежат полезные функции (например, ходилки, проверки лагов), которые используются в разных скриптах. Из этой папки _не создаются_ отдельные макросы.
- 📂 **`dist/`** — **Готовые макросы (НЕ РЕДАКТИРОВАТЬ!).** Сюда сборщик складывает готовые `.oajs` файлы. Эту папку нужно указать в настройках Orion. _Если **ты изменишь** код прямо здесь, при следующей сборке он удалится!_

## 🛠 Как превратить TypeScript в готовый макрос

Чтобы превратить исходники из папки `src` в понятные Ориону скрипты в папке `dist`, **используй** команды в терминале:

- **Разовая сборка:**

```bash
npm run build
```

- **Автоматическая сборка (Рекомендуется):**

```bash
npm run watch
```

**\*Запусти** эту команду и **сверни** терминал. Теперь каждый раз, когда **ты нажимаешь** "Сохранить" в VS Code, макросы будут мгновенно пересобираться сами!\*

## 💡 Правила написания кода

1. **Экспорт функций:** Чтобы Orion увидел **твою** функцию (например, `Autostart`), перед ней обязательно нужно написать слово `export`:

```typescript
export function Autostart() { ... }
```

2. **Типизация id и графики:** Чтобы избежать опечаток, все hex-коды **пропускай** через специальные функции-проверки. Если **ты ошибешься** (например, **забудешь** один символ), Orion напишет об этом красным цветом при запуске скрипта:

```typescript
import { toSerial, toGraphic } from '@lib/validators';

const CHEST = toSerial('0x403853AB');
const INGOTS = toGraphic('0x1BEF|0x1BE3');
```

## 🔐 Секретные настройки (Телеграм бот и пароли)

В коде макросов не хранятся реальные токены от ботов или ID чатов. Если **ты зальешь** свой токен в общую ветку, им сможет воспользоваться кто угодно! Для хранения таких данных используются "переменные окружения".

**Как настроить ключи у себя:**

1. **Найди** в корне проекта файл `.env.example`.
2. **Скопируй** его и **назови** копию просто `.env` (с точкой в начале).
3. **Открой** файл `.env` и **впиши** туда свои реальные токены и настройки:

```env
TG_BOT_TOKEN=Твой_Реальный_Токен_От_BotFather
TG_CHAT_ID=@твой_канал
TG_THREAD_ID=123
```

## 🚀 Деплой (Копирование в Orion)

Тебе не нужно вручную копировать файлы из папки `dist` в клиент игры! Мы настроили автоматический перенос.

**Как настроить:**

1. Открой файл `.env`
2. Впиши путь к папке скриптов твоего профиля в переменную `ORION_SCRIPTS_PATH`.
   _(Пример: `C:/Games/Ultima Online/Orion Launcher/OrionUO/Profiles/Main/Scripts`)_

**Как использовать:**

- Если хочешь просто скопировать уже собранные скрипты:

```bash
npm run copy
```

- Если хочешь собрать код с нуля и сразу отправить в Orion (самая удобная команда):

```bash
npm run deploy
```

## 🤝 Совместная работа (Git)

Мы работаем в команде, поэтому код хранится в общем репозитории. Чтобы мы не затирали макросы друг друга, **придерживайся** простых правил.

**Главное правило:** Мы никогда не пишем код напрямую в главную ветку (`master`). Для любой задачи создается своя ветка!

### Порядок работы:

**Шаг 1. Получить свежие обновления от команды**
Перед началом работы **убедись**, что у **тебя** самая свежая версия кода. Находясь в ветке `master`, **выполни**:

```bash
git pull
```

**Шаг 2. Создать свою ветку для работы**
**Придумай** понятное название для **твоей** задачи (например, `fix-lumber` или `new-pvp-macro`):

```bash
git checkout -b название-твоей-ветки
```

**Шаг 3. Внести изменения и сохранить их**
**Напиши** код, **проверь**, что он собирается (`npm run build`). Если всё ок, сохраняем изменения в Git:

```bash
git add .
git commit -m "Добавил авто-хил в макрос майнинга"
```

_(**Пиши** понятные комментарии к коммиту, чтобы другие понимали, что **ты сделал**)._

**Шаг 4. Отправить ветку на сервер**

```bash
git push -u origin название-твоей-ветки
```

**Шаг 5. Создать Pull Request (Запрос на слияние)**
**Зайди** на сайт GitHub (или GitLab/Bitbucket, где лежит **твой** проект). Там появится кнопка **"Compare & pull request"**. **Нажми** её.
Это создаст запрос: _"Ребята, я написал новый код, проверьте и добавьте его в главную версию"_. После проверки **твой** код будет слит с веткой `master`.
